React + TypeScript CodeMirror 6 Express + Supabase Claude API

Zhiping Zhang  ·  Qiushi Liang

CS7180 — Project 2

The Problem

Data analysts write plans as Python comments before coding:

# Step 1: Load the dataset from CSV # Step 2: Clean missing values # Step 3: Filter rows where age > 18 # Step 4: Compute summary statistics # Step 5: Visualize the distribution of income import pandas as pd # ... then manually write all the code ... pass
  • Every comment needs to be manually translated into code
  • Repetitive, breaks the analyst's flow of thinking
  • What if the editor could understand these comments?

The Solution

Click a comment line → Claude classifies it → Take action

How it works

  • Click on any # comment line in the editor
  • Claude API classifies: instruction or not?
  • If instruction → show 3 action buttons
  • Generated content inserted inline

Three Actions

  • Write Code — Generate Python implementation
  • Detail Plan — Expand into sub-steps
  • Alternative Plan — Suggest different approach

Architecture

Frontend — React + CodeMirror 6 + Tailwind CSS
POST /auth/register, /auth/login → Express Backend → Supabase users
POST /files/upload, GET /files → Express Backend → Supabase Storage + files
POST /files/:id/scan → Express Backend → Claude API + scans
POST /api/scan-line → Vercel Serverless → Claude API
POST /api/generate → Vercel Serverless → Claude API
TypeScript Strict JWT + bcrypt (cost 12) Supabase RLS claude-sonnet-4-6 Confidence ≥ 0.6

Design Evolution

Original Mockup

  • Run Scan button — batch scan entire file
  • Right sidebar listing all instructions
  • History and Settings tabs
  • Instruction types: Run, Modify, Delete, Generate, Other

Final Design

  • Click-to-detect — per-line, real-time
  • Inline action buttons below the comment
  • No sidebar, no tabs — minimal UI
  • Three actions: Write Code, Detail Plan, Alternative

Two layers of intelligence: batch scanning for overview (backend) + per-line detection for interactive editing (frontend)

The .cursorrules File

A configuration file that tells Cursor AI how to write code for our project:

Conventions

  • React components: PascalCase
  • Functions/variables: camelCase
  • API routes: kebab-case
  • DB → API: snake_case → camelCase

Rules

  • LLM: claude-sonnet-4-6 only
  • Confidence threshold: 0.6
  • TDD, 70% coverage minimum
  • Always test unauthenticated requests

Think of it as the project's constitution — every AI-generated line of code follows these rules.
